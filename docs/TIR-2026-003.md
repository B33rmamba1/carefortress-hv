## Executive Summary

Between July 22-23, 2026, two independent threat actors attempted to use stolen AWS credentials to access Amazon Bedrock AI models. This represents an emerging shift in cloud credential abuse from traditional targets (crypto mining, spam infrastructure) to AI compute theft. The stolen credentials were canary tokens served by the CareFortress honeypot and acquired through a credential marketplace — neither attacker ever contacted the honeypot directly.

## Why Bedrock?

Amazon Bedrock provides API access to foundation AI models including Anthropic Claude, Meta Llama, Amazon Titan, and Stability AI's image models. Legitimate access costs $0.25-$75+ per 1,000 requests depending on the model.

Stolen AWS credentials with Bedrock permissions provide:
- Free access to expensive AI models (Claude Opus at $75/MTok output)
- No usage limits tied to their own billing
- Anonymity — API calls appear under the victim's account
- Access to models that may have geographic or compliance restrictions

This mirrors the crypto mining trend of 2020-2023, where stolen cloud credentials were used for free compute. AI inference is now more valuable per compute-hour, making Bedrock access a higher-value target than EC2 instances for mining.

## Detection Recommendations

1. **Alert on any Bedrock API call from unexpected IPs or IAM users.** Legitimate Bedrock usage is typically limited to specific application roles. Any GetCallerIdentity followed by ListFoundationModels is a credential validation pattern.

2. **Monitor for no-user-agent AWS API calls.** The WARP attacker deliberately strips the user agent. Legitimate SDK usage always includes a user agent string. Bare API calls are a strong signal for custom exploitation tools.

3. **Enable Bedrock model invocation logging.** CloudTrail logs Bedrock control plane calls by default, but InvokeModel data events require explicit configuration. Enable them to capture the actual prompts submitted with stolen credentials.

4. **Set Bedrock service control policies.** If your organization doesn't use Bedrock, deny all Bedrock actions at the Organizations level. This prevents stolen credentials from accessing AI models even if IAM permissions would allow it.
